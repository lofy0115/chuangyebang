# 创业帮 App 开发计划

>核心定位：**从消费者抱怨中发现新价值需求，落地到可执行的商业模式**

---

## 一、产品定位与核心价值

### 1.1 目标用户
| 用户群体 | 使用场景 |
|---------|---------|
| 早期创业者 | 寻找创业切入点，验证想法 |
| 产品经理/创新负责人 | 发现新价值需求，寻找产品机会 |
| 商业模式设计师/咨询师 | 快速生成和调整商业模式方案 |
|投资人 | 分析赛道机会和潜在标的 |

### 1.2 核心痛点解决的
- **信息过载**：全网评论/反馈散落，无法系统化分析
- **缺乏结构**：有数据但不知道如何归类、提炼价值点
- **落地困难**：知道痛点但不知道如何设计商业模式
- **手工低效**：传统方式靠人工阅读、归纳，耗时数周

### 1.3 产品差异化
| 传统方式 | 创业帮 |
|---------|--------|
| 人工搜集整理 | AI自动爬取+分类 |
| 模糊的需求描述 | 结构化的需求类型+占比 |
| 凭经验设计模式 | 经典方法论+AI辅助生成 |
| 静态报告 | 可交互的参数化设计 |

```

─────────────       ───────────── ────────────── ────────────

## 三、技术架构

### 3.1 技术选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **移动端** | Flutter + Provider/Bloc | 跨平台iOS/Android |

## 四、数据模型设计

### 4.1 核心数据实体

#### 抱怨类型分类体系（二级）
```
抱怨大类
├── 性能问题（慢、卡顿、耗电）
├── 功能缺失（没有XX功能）
├── 价格问题（太贵、性价比低）
├── 服务体验（客服差、包装问题）
├── 质量缺陷（容易坏、质量差）
├── 安全隐患（质量问题、信息安全）
├── 交付问题（延迟、缺货）
└── 其他
```

#### 客户画像标签体系
```
基础属性：年龄、性别、收入、地区、职业
行为特征：消费频次、价格敏感度、品牌忠诚度
价值偏好：高端/性价比/尝鲜/实用
痛点敏感：质量>价格>服务>功能
```

#### 价值需求分类
```
功能型需求：解决具体问题
体验型需求：使用过程更便捷/愉悦
情感型需求：身份认同/社交货币
成本型需求：降低使用成本/门槛
安全型需求：放心/可靠/隐私保护
```

### 4.2 数据库表结构（关键表）

| 表名 | 用途 |
|-----|------|
| `industries` | 行业/品类分类表 |
| `raw_comments` | 原始评论爬取表 |
| `complaint_types` | 抱怨类型定义表 |
| `classified_complaints` | 已分类抱怨表 |
| `customer_profiles` | 客户画像表 |
| `value_needs` | 价值需求表 |
| `business_models` | 商业模式模板表 |
| `user_projects` | 用户项目/分析记录表 |

## 五、核心功能模块

### 5.1 数据爬取模块

| 功能 | 说明 |
|-----|------|
| **多源爬虫** | 淘宝/京东评论、微博、小红书、知乎、贴吧、论坛 |
| **智能调度** | 按行业关键词自动规划爬取任务 |
| **反反爬策略** | 代理IP轮换、请求间隔随机化、UA伪装 |
| **数据清洗** | 去重、标准化、异常值过滤 |
| **增量更新** | 定期重新爬取，追踪变化 |

### 5.2 AI分析引擎

| 功能 | 说明 |
|-----|------|
| **抱怨自动分类** | 用NLP模型将评论归类到预设类型 |
| **情感打分** | 0-100分，-5到+5情感倾向 |
| **主题聚类** | 用向量数据库聚类相似抱怨 |
| **占比统计** | 各类型抱怨的占比自动计算 |
| **客户细分** | 按标签组合细分不同客户群 |
| **需求优先级** | 按出现频次×情感强度排序 |

### 5.3 商业模式设计模块

#### 精益画布（源自《精益创业》）
```
┌─────────────┬──────────────┬─────────────┐
│ 问题 │ 解决方案 │  独特价值主张 │
│ 1. Top3痛点 │1. Top3功能  │  你的打动人心 │
│ 2. Top3痛点 │  2. Top3功能  │  的一句话描述 │
│ 3. Top3痛点 │  3. Top3功能  │             │
├─────────────┼──────────────┼─────────────┤
│ 关键指标    │  门槛优势 │  渠道 │
│ 做什么 │  技术/资源 │  触达用户路径 │
└─────────────┴──────────────┴─────────────┘
```

#### 商业模式画布（源自《商业模式新生代》）
```
┌────────────┬──────────────┬────────────┐
│ 客户细分 │  价值主张     │  渠道 │
│ 谁是客户 │  解决什么问题 │  如何触达   │
├────────────┼──────────────┼────────────┤
│ 客户关系   │  核心资源     │  关键活动 │
│ 如何维护   │  技术/人力   │  做什么     │
├────────────┼──────────────┼────────────┤
│ 成本结构   │  合作伙伴 │  收入来源   │
│ 固定/可变  │  谁帮我们 │  如何赚钱 │
└────────────┴──────────────┴────────────┘
```

#### 智能辅助功能
- **自动建议**：基于分析结果自动填充画布字段
- **参数调整**：用户可修改任何假设参数
- **快速原型**：一键生成多种备选模式方案
- **历史记录**：保存每次设计版本，对比迭代

### 5.4 用户交互模块

| 功能 | 说明 |
|-----|------|
| **行业搜索** | 输入行业/产品关键词，一键开始分析 |
| **分析仪表盘** | 抱怨类型分布饼图、客户细分矩阵 |
| **画布编辑器** | 拖拽式填写精益画布/商业模式画布 |

| **报告导出** | PDF/PPT格式完整分析报告 |
| **协作分享** | 链接分享，团队协同 |

## 七、开发阶段规划

### Phase 1：MVP核心功能（4-6周）
```
目标：验证核心价值，爬取+分析+基础展示

✅ 爬虫系统
   - 支持3个数据源（淘宝/微博/知乎）
   - 反反爬基础策略
   - 定时增量更新

✅ NLP分析引擎
   - 中文分词+情感分析
   - 抱怨类型分类（预设8类）
   - 占比统计

✅ 基础展示
   - Web端演示页面
   - 饼图+柱状图展示
   - 导出CSV

❌ 不做：商业模式画布、小程序
```

### Phase 2：分析能力增强（4周）
```
目标：提升分析深度，增加客户细分

✅客户画像系统
   - 自动识别评论中的用户特征
   - 标签化客户分类

✅ 向量聚类
   - Milvus向量数据库
   - 相似抱怨自动聚类
   - 发现长尾抱怨类型

✅ 看板优化
   - 客户细分矩阵
   - 需求优先级排序
   - 时间趋势分析
```

### Phase 3：商业模式模块（4周）
```
目标：完整的商业模式设计工具

✅ 精益画布编辑器
   - 拖拽式填写
   - 自动建议填充

✅ 商业模式画布
   - 9宫格完整版
   - 参考《商业模式新生代》模板

✅ 智能辅助
   - 基于分析结果自动生成初稿
   - 参数调整实时预览
   - 多方案对比
```

### Phase 4：移动端+协作（3周）
```
目标：移动端App，协作分享

✅ Flutter App
   - iOS/Android双平台
   - 核心分析功能

✅ 协作功能
   - 链接分享
   - 评论讨论

✅ 报告导出
   - PDF完整报告
   - PPT演示版
```

### Phase 5：商业化探索（持续）
```
目标：验证商业模式

- 免费基础功能
- 付费高级分析（更多数据源、深度报告）
- 企业版（团队协作、品牌分析）

## 八、《精益创业》核心思路落地

| 精益创业概念 | 在创业帮中落地 |
|-------------|--------------|
| **MVP（最小可行产品）** | Phase 1只做核心爬虫+分析，不做完整App |
| **验证性学习（Validated Learning）** | 用户分析结果=假设验证的证据 |
| **创新核算（Innovation Accounting）** | 追踪"有多少用户完成分析→设计" |
| **转型vs坚持（Pivot or Persevere）** | 画布调整后追踪指标变化，验证假设 |
| **构建-测量-学习循环** | 搜索→分析→设计→验证→迭代 |

## 十、技术难点与解决方案

| 难点 | 解决方案 |
|-----|---------|
| **反爬虫对抗** | 代理IP池+分布式爬虫+请求随机化 |
